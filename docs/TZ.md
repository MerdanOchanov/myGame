# Техническое задание: Scientists World

## 1. Суть проекта

Scientists World — геолокационная web-игра на реальной карте мира, где игроки выступают в роли ученых, физически исследуют биомы, собирают природные материалы, создают неизвестные лекарства в доме-лаборатории и выживают под воздействием глобальных и локальных событий.

Игра рассчитана на desktop и mobile browser с интеграцией реальной карты и геопозиции игрока.

## 2. Основная идея

Игроки из любой страны мира становятся учеными-исследователями. Первое сообщество и контент могут стартовать с Туркменистана, но архитектура и карта сразу проектируются как world-first.

1. Мир игры основан на реальной карте.
2. Поверх карты накладываются процедурные игровые слои: биомы, hex-ячейки, дома-лаборатории игроков, локальные события.
3. Игрок может исследовать биомы только при физическом присутствии в координатах соответствующей зоны.
4. Материалы собираются из реальных координатных зон: растения, фрукты, ягоды, насекомые.
5. Все материалы имеют процедурно сгенерированные свойства.
6. Лекарства создаются смешением 3 и более материалов.
7. Эффекты лекарства заранее неизвестны и должны тестироваться на крысах.
8. Игроки выживают за счет правильного исследования, экспериментов, обмена и применения лекарств.
9. Главный рейтинг — `Top-100` по количеству реальных дней, прожитых без смерти.

Ключевая идея: не просто собрать ресурсы и сделать предмет, а научно исследовать неизвестные свойства мира и превращать их в лекарства для выживания.

## 4. Карта мира и hex-сетка

Игровое поле строится на реальной карте мира.

Поверх карты должны быть игровые слои:

| Слой | Назначение |
|---|---|
| Real map | Базовая карта мира |
| Hex grid | Игровые ячейки диаметром 50 метров |
| Procedural biomes | Процедурные биомы поверх реальных координат |
| Player homes | Дома-лаборатории игроков в реальном времени |
| Events | Глобальные и локальные события |
| Resource availability | Доступность материалов в зоне |

Hex-ячейка:

1. Форма — шестигранник.
2. Базовый диаметр — 50 метров.
3. Используется для дома-лаборатории, биомов, событий и спавна материалов.
4. Координаты игрока должны маппиться в конкретную `hexCellId`.

```ts
interface HexCell {
  id: string;
  center: { lat: number; lng: number };
  diameterMeters: 50;
  biomeId?: string;
  ownerPlayerId?: string;
}
```

## 5. Дом-лаборатория

При первом входе игрок может закрепить за собой точку на карте как дом-лабораторию.

Требования:

1. Дом выбирается при первом входе.
2. Дом занимает одну hex-ячейку диаметром 50 метров.
3. Дом привязан к реальным координатам.
4. Дом отображается на карте домов игроков в реальном времени только для игроков в радиусе 1000 метров.
5. Дом является лабораторией для крафта лекарств и тестирования на крысах.
6. Дом нельзя создать поверх занятой hex-ячейки.
7. Дом можно переносить по отдельному игровому правилу.
8. Мировые и локальные события могут обнулять дома в затронутых hex-зонах.

```ts
interface LaboratoryHome {
  id: string;
  playerId: string;
  hexCellId: string;
  position: { lat: number; lng: number };
  claimedAt: string;
  diameterMeters: 50;
  level: number;
}
```

Функции дома:

| Функция | Описание |
|---|---|
| Crafting | Смешение материалов и создание лекарств |
| Rat testing | Проверка неизвестных эффектов лекарства |
| Storage | Хранение материалов, лекарств и результатов тестов |
| Research log | История экспериментов |
| Map anchor | Постоянная точка игрока на карте |

## 6. Биомы

Биомы процедурно генерируются поверх реальной карты.

Размер биома:

1. Биом состоит из 3-6 hex-зон.
2. Каждая hex-зона имеет диаметр 50 метров.
3. Итоговый биом — компактная группа соседних hex-ячеек.

```ts
interface Biome {
  id: string;
  type: BiomeType;
  hexCellIds: string[];
  generatedAt: string;
  seed: string;
}
```

Примеры типов биомов:

| Биом | Материалы | Особенности |
|---|---|---|
| Desert | сухие растения, насекомые, минералы | высокая жара, дефицит воды |
| Steppe | травы, ягоды, мелкие насекомые | стабильная зона сбора |
| Oasis | фрукты, ягоды, водные растения | редкие лечебные свойства |
| Mountain | корни, мхи, редкие насекомые | сложный доступ |
| Urban fringe | синантропные растения, лабораторный мусор | нестабильные свойства |
| Coast | водоросли, соль, прибрежные растения | водные эффекты |

Исследование биома:

1. Игрок должен физически находиться внутри hex-ячейки биома.
2. Клиент отправляет координаты.
3. Сервер проверяет допустимость координат.
4. Игрок получает доступ к сбору материалов только в текущем биоме.
5. Изучение биома постепенно раскрывает список найденных материалов.

## 7. Материалы

Материалы — растения, фрукты, ягоды и насекомые.

Категории:

| Категория | Примеры |
|---|---|
| Plants | травы, корни, листья, цветы |
| Fruits | плоды деревьев и кустарников |
| Berries | ягоды |
| Insects | жуки, личинки, пауки, муравьи |

Каждый материал имеет 20+ процедурно сгенерированных характеристик.

Главное правило генерации:

1. Материал генерируется случайно только один раз.
2. После первой генерации материал фиксируется навсегда.
3. Название, характеристики, проценты, биом и seed сохраняются в базе.
4. Все игроки, нашедшие этот материал позже, получают ту же зафиксированную версию.

```ts
interface Material {
  id: string;
  name: string;
  category: 'plant' | 'fruit' | 'berry' | 'insect';
  biomeType: string;
  originHexCellId: string;
  generationSeed: string;
  primaryTraits: MaterialTrait[];
  secondaryTraits: MaterialTrait[];
  discoveredAt: string;
  discoveredByPlayerId?: string;
}

interface MaterialTrait {
  effectKey: string;
  percent: number;
  visibility: 'hidden' | 'partially_known' | 'known';
}
```

Характеристики материалов:

| Тип | Описание |
|---|---|
| Акцентные | Основные свойства, сильнее влияют на итоговое лекарство |
| Дополнительные | Побочные и слабые свойства |
| Скрытые | Неизвестны игроку до тестирования |
| Изученные | Подтверждены экспериментами |

Пример характеристик:

| Характеристика | Влияние |
|---|---|
| `anti_fever` | Снижает жар |
| `toxic` | Повышает отравление |
| `sedative` | Вызывает сонливость |
| `stimulant` | Повышает активность |
| `anti_infection` | Борется с инфекцией |
| `hydration` | Влияет на водный баланс |
| `pain_relief` | Снижает боль |
| `mutation_risk` | Может создать непредсказуемый эффект |

## 8. Состояния и эффекты

У крыс и игроков должно быть 100+ состояний и эффектов.

Состояния — это измеримые параметры организма.

Эффекты — временные изменения, которые накладываются событиями, лекарствами или болезнями.

```ts
interface BiologicalState {
  key: string;
  value: number;
  min: number;
  max: number;
}

interface ActiveEffect {
  key: string;
  power: number;
  durationSec: number;
  source: 'event' | 'medicine' | 'disease' | 'biome';
}
```

Примеры групп состояний:

| Группа | Примеры |
|---|---|
| Vital | health, stamina, hydration, body_temperature |
| Disease | infection, fever, inflammation, parasite_load |
| Mental | stress, panic, focus, hallucination |
| Toxicology | poison, radiation, allergy, liver_load |
| Immunity | immunity, resistance, recovery_speed |
| Mobility | speed, fatigue, muscle_pain |
| Sensory | vision, hearing, smell |
| Experimental | mutation, unknown_reaction, instability |

## 9. Крысы и тестирование лекарств

Крысы — основной инструмент лабораторных испытаний.

Назначение:

1. Проверять неизвестные лекарства до применения на игроке.
2. Открывать скрытые эффекты материалов.
3. Давать статистику риска.
4. Создавать научную часть игрового процесса.

```ts
interface LabRat {
  id: string;
  ownerPlayerId: string;
  state: BiologicalState[];
  activeEffects: ActiveEffect[];
  alive: boolean;
  createdAt: string;
}
```

Процесс теста:

1. Игрок создает лекарство из 3+ материалов.
2. Лекарство получает неизвестный набор эффектов.
3. Игрок выбирает крысу для теста.
4. Система применяет лекарство к крысе.
5. Через результат теста открываются часть эффектов лекарства.
6. При негативном исходе крыса может умереть.
7. Игрок решает, применять ли лекарство на себе или обменивать его.

## 10. Крафт лекарств

Крафт ограничен только лекарствами.

Правила:

1. Лекарство создается из 3 и более материалов.
2. Материалы могут быть разных категорий.
3. Шанс успешного крафта случайный, но зависит от характеристик материалов, изученности рецепта и уровня лаборатории.
4. Итоговые эффекты лекарства зависят от процентных характеристик материалов.
5. Название и свойства лекарства могут быть неизвестны до тестов.
6. Повторение той же комбинации материалов должно давать воспроизводимый результат при одинаковом seed и версии правил.

```ts
interface Medicine {
  id: string;
  name: string;
  creatorPlayerId: string;
  inputMaterialIds: string[];
  generationSeed: string;
  knownEffects: ActiveEffect[];
  hiddenEffects: ActiveEffect[];
  successChance: number;
  stabilityPercent: number;
  createdAt: string;
}
```

Алгоритм крафта:

1. Игрок выбирает минимум 3 материала.
2. Система проверяет, что игрок находится в доме-лаборатории.
3. Система рассчитывает `successChance`.
4. При провале материалы списываются частично или полностью.
5. При успехе создается лекарство.
6. Эффекты лекарства рассчитываются из характеристик материалов.
7. Часть эффектов остается скрытой.
8. Игрок тестирует лекарство на крысах для раскрытия эффектов.

## 11. Инвентарь

Инвентарь игрока бесконечный.

Он содержит:

1. Собранные материалы.
2. Готовые лекарства.
3. Результаты исследований.
4. Историю тестов.

```ts
interface Inventory {
  playerId: string;
  materials: InventoryStack[];
  medicines: InventoryStack[];
}

interface InventoryStack {
  itemId: string;
  quantity: number;
}
```

Ограничения веса в MVP не нужны.

## 12. Физический бартер

Бартер нужен для обмена лекарствами и материалами между игроками. Это не глобальный магазин из меню, а физическая встреча в случайной hex-ячейке.

Экономика — бартерная, без валюты на старте.

```ts
interface BarterMeeting {
  id: string;
  initiatorPlayerId: string;
  targetPlayerId?: string;
  meetingHexCellId: string;
  center: { lat: number; lng: number };
  offeredItems: BarterItem[];
  requestedItems: BarterItem[];
  status: 'created' | 'waiting' | 'ready' | 'completed' | 'cancelled' | 'expired';
  createdAt: string;
  expiresAt: string;
}

interface BarterItem {
  itemId: string;
  itemType: 'material' | 'medicine';
  quantity: number;
}
```

Требования:

1. Игрок создает barter-заявку с предложением и желаемым обменом.
2. Система выбирает случайную hex-ячейку в радиусе 500 метров.
3. Для обмена оба игрока должны физически присутствовать в этой hex-ячейке.
4. Обмен происходит только если обе стороны имеют нужные предметы.
5. В интерфейсе обмена видны только известные конкретному игроку свойства товара.
6. Скрытые свойства лекарства не раскрываются автоматически.
7. Если игроки не пришли в точку до `expiresAt`, встреча истекает.

## 13. События

Динамика игры создается мировыми и локальными событиями.

| Тип события | Описание | Пример |
|---|---|---|
| Global | Действует на всех игроков | Пандемия, жара, магнитная буря |
| Regional | Действует на страну или регион | Пыльная буря в Туркменистане |
| Local | Действует на группу hex-ячеек | Загрязнение биома, вспышка инфекции |
| Personal | Действует на одного игрока | Аллергия, заражение, усталость |

События накладывают эффекты на игроков.

```ts
interface GameEvent {
  id: string;
  scope: 'global' | 'regional' | 'local' | 'personal';
  affectedHexCellIds?: string[];
  affectedRegionId?: string;
  effects: ActiveEffect[];
  startsAt: string;
  endsAt: string;
}
```

Роль лекарств:

1. Снижать вред событий.
2. Убирать негативные эффекты.
3. Давать временную защиту.
4. Иногда создавать побочные эффекты.

## 14. Смерть и рейтинг

Главная метрика рейтинга — количество реальных дней, прожитых без смерти.

```ts
interface SurvivalRun {
  playerId: string;
  startedAt: string;
  endedAt?: string;
  alive: boolean;
  survivedRealDays: number;
}
```

Правила:

1. При первом входе начинается survival-run.
2. Если игрок умирает, текущий run завершается.
3. Игрок может начать новый run.
4. В `Top-100` попадают игроки с максимальным количеством реальных дней без смерти.
5. Рейтинг должен учитывать только подтвержденные сервером данные.
6. При смерти игрок теряет текущий survival-run, инвентарь и лекарства.
7. Судьба дома после смерти определяется отдельным правилом баланса; события мира могут обнулять дом независимо от смерти.

## 15. Multiplayer и реальное время

MVP сразу делается с backend, потому что есть:

1. Карта домов игроков рядом с текущим игроком.
2. Фиксация процедурно сгенерированных материалов навсегда.
3. Проверка геопозиции.
4. События мира.
5. Физический бартер.
6. Рейтинг Top-100.

Требования к реальному времени:

| Данные | Обновление |
|---|---|
| Дома игроков в радиусе 1000 м | Почти real-time |
| Локальные события | Real-time или polling |
| Barter-встречи | Near real-time |
| Рейтинг | Периодическое обновление |
| Позиция игроков | Не показывать всем постоянно в MVP |

Архитектурный стек backend пока не фиксируется. Это отдельное решение, но сам backend обязателен для MVP.

## 16. MVP

MVP должен доказать основной цикл: реальная карта, геопозиция, hex-дом, биом, сбор материала, крафт лекарства, тест на крысе.

Минимальный сценарий:

1. Игрок открывает игру.
2. Браузер запрашивает геопозицию.
3. В debug-режиме разработчик может выбрать позицию кликом по карте.
4. Игрок видит карту.
5. Backend подтверждает текущую hex-ячейку игрока.
6. Игрок занимает свободную hex-ячейку диаметром 50 метров как дом-лабораторию.
7. Игрок видит текущий процедурный биом.
8. Игрок физически находится в зоне биома или использует debug-позицию в тестовом режиме.
9. Игрок собирает минимум 3 материала.
10. Материалы получают фиксированные процедурные характеристики.
11. Игрок создает неизвестное лекарство.
12. Игрок тестирует лекарство на крысе.
13. Система раскрывает часть эффектов лекарства только для этого игрока.
14. Игрок может применить лекарство к себе.

## 19. Критерии готовности MVP

MVP считается готовым, если:

1. Игра запускается на desktop и mobile browser.
2. Карта отображает реальный регион игрока.
3. Браузер получает геопозицию игрока.
4. Координаты игрока переводятся в hex-ячейку.
5. Игрок может закрепить свободную hex-ячейку 50 м как дом-лабораторию.
6. На карте отображается дом игрока.
7. Биом состоит из 3-6 соседних hex-ячеек.
8. Игрок может собирать материалы только при физическом присутствии в биоме.
9. Материал получает 20+ характеристик при первой генерации.
10. Сгенерированный материал фиксируется и повторно не меняется.

14. У крысы есть набор состояний и активных эффектов.
15. Тест раскрывает часть свойств лекарства.
